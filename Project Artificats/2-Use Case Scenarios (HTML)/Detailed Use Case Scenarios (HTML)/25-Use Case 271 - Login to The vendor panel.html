89 - Login to The Vendor Panel

Primary Actors

  • Vendor

Preconditions

  1. Vendor account already exists.

Basic Flow (Happy Path)

  1. Vendor navigates to the home page.
  2. Vendor clicks on the Sign In link.
  3. The login page shows username and password input fields, a Remember Me checkbox, a Reset Password link and a Create Account link is displayed.
  4. Vendor enters his username and his password.
  5. Vendor checks the Remember me button.
  6. Vendor clicks on the login button.
  7. The vendor credentials are validated.
  8. Vendor is redirected to Vendor panel.

Postconditions

  1. The vendor has been logged in successfully.
  2. Vendor is kept logged in, so for his next visit to the site he won't need to re-enter his username and password.

Alternative Flow

7.1 Invalid Username or Password

if the username or password provided by vendor is invalid, then

7.1.1. An error message will show indicating that the username or password is invalid.

7.1.2 The use case resumes at step 3.
