2116 - Provide paging for products listing

Primary Actors

  • Visitor

Preconditions

  1. Vendors have already added at least 81 products.

Basic Flow (Happy Path)

  1. Visitor navigates to the home page and clicks on view all link from the new arrivals slider.
  2. A page shows first 20 new arrivals products is displayed.
  3. The page has a paging area where visitor can navigate between different pages which shows the rest of the products.
  4. The paging area shows buttons to navigate to the first page, previous page, current page, four buttons for each of the next four pages, next page and last page respectively.
  5. The first page, previous page and current page buttons are disabled.
  6. The visitor clicks on the second page button.
  7. The list is updated with other products in the second page.
  8. The first page and previous page buttons are now enabled.
  9. The current page button is still disabled.

Postcondition

None

Alternative Flow

2.1 The number of products per page is set to 30

if the settings were modified to show 30 products per page, then

2.1.1. The page will show 30 products instead of 20.

2.1.2. The use case resumes at step 3.
