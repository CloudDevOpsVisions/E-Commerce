143 - Update completed order

Primary Actors

  • Vendor

Preconditions

  1. Vendor has been logged to his account.
  2. At least one order was made to the vendor.

Basic Flow (Happy Path)

  1. Vendor clicks on a specific order from list all orders page.
  2. A page shows order details which is the Quantity of products in order, Price of each product in order, Total price for products in order, Order Status and Gift wrap option for order is displayed.
  3. The member's personal information which is First name, Last name, Email, Phone, City, State, Postal code, Country and Address of shipping details is also displayed.
  4. The vendor clicks on the member's personal information area.
  5. A dialog shows the member's personal information into input fields and a save button is displayed.
  6. The vendor corrects a spelling mistake in the member's first name field.
  7. The vendor clicks on the save button.
  8. The member information is validated.
  9. The dialog disappears.

Postcondition

  1. The order information is updated.

Alternative Flow

8.1 Invalid first name

if vendor provides an invalid first name, then

8.1.1. An error message shows to indicate that the first name provided is invalid.
