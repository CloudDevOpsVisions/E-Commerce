202 - Display/hide special offer for products

Primary Actors

  • Admin

Preconditions

  1. Admin has been logged successfully to his account.
  2. Admin has been authorized to change the displayed offers.
  3. Vendors have already added at least three offers.
  4. No special offers are selected.

Basic Flow (Happy Path)

  1. Admin navigates to the admin panel.
  2. Admin clicks on the special offers link.
  3. The special offers list page is displayed, showing the first 20 offers in rows ordered by recent start date.
  4. Each row shows a checkbox, vendor name, offer title, product name, offer start date, offer end date, offer price and discount.
  5. There is a paging area where admin can navigate between different pages which displays the rest of the offers.
  6. There is a save button which is gray (unavailable) indicating that there are no changes to be saved.
  7. Admin checks 3 offers, each of which is highlighted once checked.
  8. The save button becomes blue (available) indicating that there are changes to be saved.
  9. Admin clicks on the save button.
  10. Admin choices are validated.
  11. The save button becomes gray (unavailable) indicating that the changes have been saved.
  12. The list is updated reordered by checked offers then by recent start date.

Postconditions

  1. The special offers slider in home page is updated with the three offers chosen by admin.

Alternative Flow

10.1 Admin checked 4 offers

if admin checks 4 offers, then

10.1.1. An error message will show indicating that the maximum number of special offers allowed is 3.

10.1.2 The use case resumes at step 3.

10.2 An Offer end date has already passed

if admin checks an offer which has an end date that already passed, then

10.2.1. And error message will show indicating that one or more of the selected offers has already ended.

10.2.2 The offer with past date is unchecked and unhighlighted.

10.2.3 The use case resumes at step 3.
